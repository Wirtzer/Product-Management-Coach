# Builder's Credibility — Lesson Plan

**Updated:** 2026-01-01

## Current Status
New learner. No sessions completed yet. Begin with diagnostic to establish baseline.

## What to Cover First
1. **Code reading for PMs** — Learn to read code well enough to understand system architecture, identify bottlenecks, and ask informed questions in design reviews. Focus on reading, not writing — the goal is technical literacy, not engineering.
2. **Architecture review skills** — Learn to evaluate system designs for PM-relevant tradeoffs: scalability, latency, cost, reliability, time-to-market. Practice asking the questions engineers respect.
3. **Technical spec writing** — Write specs that engineers trust. Learn the structure: problem statement, proposed approach, alternatives considered, data model, API design, rollout plan, rollback plan.

## Recommended First Exercise
Find an open-source project you find interesting (e.g., on GitHub). Read its README and one core source file. Explain in plain language: What does this system do? How is it structured? What would you want to know before making a product decision about it? The coach will evaluate your technical reading comprehension.

## Phase Progression
- Observe → Assist → Lead → Teach → Innovate
- Current phase: Observe (read and explain technical systems)

## Key Dimensions
- Core concepts, practical application, current research, mathematical intuition
