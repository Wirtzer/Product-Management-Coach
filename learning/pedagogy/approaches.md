# Pedagogy Approaches Catalog

Reference for the PM Coach's daily reflection agent and live coaching sessions. Defines available teaching techniques, when to use them, and how to measure progress.

---

## 1. Storytelling Practice

**Scope:** LP Stories, Behavioral Answers (STAR format)
**Goal:** Fluent, vivid, structured delivery under pressure

### Phase Progression
1. **Introduction** — Identify raw material: projects, decisions, conflicts, results
2. **Structuring** — Shape into STAR; attach metrics; clarify the "so what"
3. **Practice** — Repeated delivery with feedback on clarity and flow
4. **Refinement** — Polish language, tighten timing, eliminate fillers
5. **Performance** — Deliver under realistic constraints (time pressure, curveballs)

### Techniques
| Technique | Description | Best Phase |
|-----------|-------------|------------|
| `basic-run-through` | Tell the story start to finish; coach listens and notes gaps | Introduction, Structuring |
| `record-and-review` | The learner narrates; coach plays back weak sections with specific fixes | Practice |
| `chunk-and-polish` | Isolate one STAR section (e.g., just the Result) and drill it | Refinement |
| `targeted-drill` | Rapid-fire on a known weakness (e.g., quantifying impact, opening hook) | Refinement |
| `pressure-simulation` | Timed delivery with interruptions or follow-ups. **USE WITH CAUTION** — only when the learner is in Refinement+ phase and confidence is solid. Premature pressure erodes fluency. | Performance |
| `variation-practice` | Tell the same story at 30s, 60s, and 2min lengths; adapt for different LPs | Performance |

### Mastery Signals
- STAR structure flows without conscious effort
- Numbers and impact land in the first sentence of Result
- Filler words ("um", "like", "basically") are rare
- "So what" bridge between Result and LP feels natural
- Adapts story length to interviewer cues without losing substance

## 2. Framework Application

**Scope:** CIRCLES, RICE, AARRR, Metrics Design, and similar PM frameworks
**Goal:** Internalized frameworks that deploy automatically in novel situations

### Phase Progression
1. **Learn** — Understand structure, purpose, and canonical examples
2. **Apply (Guided)** — Coach walks through application on a case together
3. **Apply (Solo)** — The learner applies independently; coach reviews after
4. **Transfer** — Apply framework to unfamiliar domains (e.g., RICE on a hardware product)
5. **Adapt** — Modify or combine frameworks for situations where none fits perfectly

### Techniques
| Technique | Description | Best Phase |
|-----------|-------------|------------|
| `worked-example` | Coach demonstrates full framework application, narrating reasoning | Learn |
| `guided-application` | The learner leads; coach prompts at decision points | Apply (Guided) |
| `solo-application` | The learner applies end-to-end; coach gives feedback after completion | Apply (Solo) |
| `compare-and-contrast` | Apply two frameworks to the same problem; discuss tradeoffs | Transfer |
| `framework-selection` | Present a problem without specifying framework; the learner picks and justifies | Adapt |

### Mastery Signals
- Applies correct framework to a novel case without prompting
- Chooses the right framework and articulates why alternatives are weaker
- Identifies situations where a framework doesn't apply or needs modification

## 3. Case Analysis

**Scope:** Product Sense, Product Strategy, Market Analysis
**Goal:** Pattern recognition and structured analytical thinking

### Phase Progression
1. **Observe** — Study expert analyses; learn what "good" looks like
2. **Analyze (Guided)** — Coach scaffolds the analytical process step by step
3. **Analyze (Solo)** — The learner drives the analysis independently
4. **Synthesize** — Connect patterns across multiple cases to derive principles
5. **Create** — Generate original strategic recommendations for real products

### Techniques
| Technique | Description | Best Phase |
|-----------|-------------|------------|
| `case-walkthrough` | Coach presents a case and walks through ideal analysis | Observe |
| `case-dissection` | Given a completed analysis, the learner identifies strengths, gaps, and assumptions | Observe, Analyze (Guided) |
| `pattern-matching` | Present 2-3 cases; the learner identifies the common strategic pattern | Synthesize |
| `reverse-engineering` | Start from an outcome (product launch, pivot); work backward to the logic | Analyze (Solo) |
| `live-case` | Analyze a real, current product situation with no prepared answer | Create |
| `blind-case` | The learner receives a case cold with no context; must structure from scratch | Create |

### Mastery Signals
- Identifies the right analytical lens (growth vs. monetization vs. retention) quickly
- Structures analysis without prompting from coach
- Spots second-order effects and downstream consequences

## 4. Concept Learning

**Scope:** Pricing, Unit Economics, Technical Concepts
**Goal:** Deep understanding that enables explanation and application

### Phase Progression
1. **Concrete Example** — Start with a specific, tangible instance
2. **Abstract Principle** — Extract the general rule from the example
3. **Application** — Apply the principle to a new situation
4. **Edge Cases** — Explore where the principle breaks down
5. **Teaching** — The learner explains the concept as if teaching someone else

### Techniques
| Technique | Description | Best Phase |
|-----------|-------------|------------|
| `concrete-first` | Begin with a real-world example before any abstraction | Concrete Example |
| `build-up` | Layer complexity incrementally on a working mental model | Abstract Principle |
| `socratic-questioning` | Guide the learner to discover principles through questions. **Only AFTER a concrete anchor is established** — premature Socratic method causes frustration without grounding. | Abstract Principle, Application |
| `analogy-bridge` | Connect new concept to something the learner already understands deeply | Application |
| `teach-back` | The learner explains the concept; coach plays a confused student | Teaching |
| `edge-case-exploration` | Present scenarios where the concept behaves unexpectedly | Edge Cases |

### Mastery Signals
- Explains the concept without jargon to a non-expert
- Applies the concept to novel situations correctly
- Handles edge cases and knows the boundaries of applicability

## 5. AI Concept Learning

**Scope:** Technical Depth — Transformers, JEPA, RL, Diffusion Models, etc.
**Goal:** Deep understanding that enables expert-level conversation

### Phase Progression
1. **Concrete Anchor** — Ground the concept in a tangible demo or use case
2. **Intuition Building** — Develop a mental model of *why* it works
3. **Formal Understanding** — Engage with the actual mechanism (architecture, math intuition, tradeoffs)
4. **Application** — Connect to product decisions and real deployments
5. **Expert Discourse** — Discuss with precision, form opinions, reference key work

### Techniques
| Technique | Description | Best Phase |
|-----------|-------------|------------|
| `concrete-anchor` | Start with a demo, product, or output the learner can experience directly | Concrete Anchor |
| `intuition-building` | Use visual metaphors and simplified models to build the "feel" | Intuition Building |
| `paper-walkthrough` | Walk through a key paper's contributions, skipping dense math, focusing on insight | Formal Understanding |
| `analogy-bridge` | Map AI concept to a domain the learner already knows (logistics, product metrics) | Intuition Building, Formal Understanding |
| `debate-practice` | Take opposing positions on an AI topic (e.g., "scaling laws will plateau") | Expert Discourse |
| `current-events-hook` | Use recent AI news as entry point to deepen technical understanding | Any phase |
| `expert-vocabulary-drill` | Practice using precise terminology in context until it feels natural | Expert Discourse |

### Mastery Signals
- Explains concept to a non-expert clearly and accurately
- Explains to an expert using correct terminology and nuance
- Knows key papers and researchers in the sub-field
- Has formed defensible opinions on open questions
- Connects technical concepts to product implications

## 6. Industry Case Analysis

**Scope:** AI Industry Landscape, Competitive Dynamics, Market Structure
**Goal:** Current, deep knowledge of the AI landscape

### Phase Progression
1. **Survey** — Map the landscape: key players, segments, funding patterns
2. **Deep Dive** — Go deep on specific companies, technologies, or market segments
3. **Pattern Recognition** — Identify recurring dynamics across the industry
4. **Synthesis** — Build a coherent mental model of where the industry is heading
5. **Forecasting** — Make and track predictions; refine judgment over time

### Techniques
| Technique | Description | Best Phase |
|-----------|-------------|------------|
| `market-mapping` | Collaboratively build a landscape map of a sub-sector | Survey |
| `competitive-analysis` | Deep dive on one company: strategy, moat, vulnerabilities | Deep Dive |
| `funding-analysis` | Trace funding rounds to understand investor thesis and market signals | Deep Dive |
| `timeline-construction` | Build a chronological narrative of how a market segment evolved | Pattern Recognition |
| `prediction-practice` | The learner makes specific, falsifiable predictions; revisit later | Forecasting |
| `news-debrief` | Discuss a recent industry event; extract the strategic signal from noise | Any phase |

### Mastery Signals
- Names the top 5 companies in a given AI sub-field from memory
- Knows recent funding rounds and what they signal
- Predicts likely next moves by key players with reasoning
- Understands talent flows and their implications

## 7. Interview Drill

**Scope:** Mock Interviews, Combined Skills Under Pressure
**Goal:** Integration of all skills under realistic interview conditions

### Phase Progression
1. **Component Practice** — Drill individual skills (story, framework, case) in isolation
2. **Partial Simulation** — Combine 2-3 skills in a semi-structured format
3. **Full Simulation** — End-to-end mock interview with realistic conditions
4. **Debrief Mastery** — Self-assess accurately; identify own gaps without coach prompting

### Techniques
| Technique | Description | Best Phase |
|-----------|-------------|------------|
| `timed-response` | Strict time limits on answers (60s, 90s, 2min) | Component Practice, Partial Simulation |
| `interviewer-pushback` | Coach challenges answers, asks "why not X?", probes weak points | Partial Simulation, Full Simulation |
| `topic-pivot` | Abrupt topic change mid-interview; tests adaptability | Full Simulation |
| `panel-simulation` | Coach plays multiple interviewer personas in sequence | Full Simulation |

### Mastery Signals
- Natural transitions between frameworks and stories without awkward pauses
- Handles curveball questions with composure and structure
- Confident under pressure; recovers gracefully from stumbles
- Self-debriefs accurately after a mock

## Universal Principles

These apply across **all** topic types. The coach and reflection agent should verify adherence in every session.

1. **Zone of Proximal Development** — Work at the edge of the learner's current ability. Too easy = boredom. Too hard = shutdown. Aim for productive struggle.
2. **Feedback Timing** — Immediate correction for factual errors. Delayed feedback (end of exercise) for structural and stylistic issues.
3. **Interleaving** — Mix topic types within and across sessions for stronger long-term retention. Avoid drilling one topic for too many sessions in a row.
4. **Desirable Difficulty** — Target a ~70-80% success rate. If the learner is succeeding 100% of the time, increase challenge. Below 60%, scaffold more.
5. **Concrete-to-Abstract** — Always start with a concrete example before introducing abstract principles. Most learners learn best when grounded first.
6. **Chain-of-Thought** — Coach explains its own reasoning when teaching, making the thinking process visible and transferable.
7. **Self-Criticism** — The reflection agent critiques its own lesson plans and session designs, identifying what could be improved before the next session.
8. **Decomposition** — Break complex topics into sub-problems. Tackle each piece, then integrate. Never present a monolith.

## Phase Advancement Criteria

A learner advances to the next phase when **all three** conditions are met:

| Criterion | Threshold | Rationale |
|-----------|-----------|-----------|
| **Quantitative** | Mastery score for the current phase's target dimensions exceeds **75/100** | Ensures baseline competence before adding complexity |
| **Qualitative** | The learner demonstrates the phase's mastery signals in **2+ consecutive sessions** | Guards against one-off performance; proves consistency |
| **Spaced Repetition** | At least one successful review after a **3+ day gap** | Confirms retention, not just short-term recall |

If any criterion is not met, remain in the current phase and adjust technique selection to address the gap.

## 8. Craft-Based Practice

**Scope:** Narrative & Influence
**Goal:** Build persuasive communication skills through iterative writing and revision

### Phase Progression
1. **Observe** — Study great product narratives from successful launches
2. **Imitate** — Write in the style of exemplar narratives
3. **Adapt** — Iterate on drafts with targeted feedback
4. **Create** — Develop original narratives for different audiences
5. **Master** — Deliver narratives live and handle Q&A

### Techniques
| Technique | Description | Best Phase |
|-----------|-------------|------------|
| `exemplar-study` | Study great product narratives | Observe |
| `imitation-exercise` | Write in style of exemplar | Imitate |
| `revision-workshop` | Iterative feedback on drafts | Adapt |
| `audience-adaptation` | Rewrite same content for different audiences | Create |
| `live-delivery` | Present narrative and handle Q&A | Master |

### Mastery Signals
- Can write compelling 1-pager without scaffolding
- Adapts framing for audience automatically
- Coach feedback is polish-level not structural
